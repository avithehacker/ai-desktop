## Architecture Overview

This is a multi-platform AI chat desktop app (Ramanujan) that ships as:
- **Electron app** — full local AI stack (Ollama, WebLLM, cloud providers)
- **Browser/web app** — same UI, routes via standalone API server
- **CLI** (`ram`) — terminal interface, same routing logic
- **VS Code extension** — inline AI assistant, same config

The core pattern is a **local-first AI router**: classify the prompt → score available models → try local first (Ollama/WebLLM) → fall back to cloud (Anthropic, OpenAI, Gemini, GitHub Models) only if needed.

## Deep Trace: `createBrowserAPI()` — The Dual-Platform Seam

`createBrowserAPI()` is the single highest-betweenness node because it's the shim that lets the same UI run in both Electron and browser mode without any component knowing which platform it's on.

```
UI Components (community 0)
  ChatArea / InputBar / Sidebar / ModelPicker
        ↓  call window.electronAPI.xxx()

ElectronAPI interface  (src/electron.d.ts)
  ← type contract both platforms must satisfy →

        ↓ Electron:                ↓ Browser:
  window.electronAPI          createBrowserAPI()  ← factory in browserAPI.ts
  injected by preload.ts      returns same interface
  main process handles it     routes HTTP to…

                        Standalone API Server  (api/src/server.ts)
                        classifyTask · tryLocal · tryCloud
                        OR: WebLLM inference directly in-browser
```

`main.tsx` calls `createBrowserAPI()` at startup when `window.electronAPI` is absent. Every UI component then calls `window.electronAPI.xxx()` identically, regardless of runtime. The factory wires up `streamToChat()` (→ `localhost:PORT/stream`) and `classify()` (→ API server) in browser mode, or falls back to in-browser WebLLM.
